# Superinstance

## The Foundational Concept

A **Superinstance** is a persistent, autonomous software entity that exists as a complete computational environment. Unlike traditional applications or services that run on demand, a Superinstance is always "on" — maintaining state, executing cycles, and interacting with its environment through defined interfaces.

### Key Characteristics

1. **Persistent Identity**: A Superinstance maintains continuity across sessions, reboots, and even migrations.
2. **Autonomous Operation**: It executes according to its internal logic without requiring constant human intervention.
3. **Environmental Integration**: It interacts with its host system, network, and other Superinstances through well-defined protocols.
4. **Stateful Evolution**: It learns, adapts, and grows over time while maintaining core identity.

### The Lucineer Implementation

In the Lucineer fleet, each vessel (like Capitaine) is a Superinstance with specific characteristics:

- **Body as Code**: The repository itself constitutes the vessel's physical form
- **Memory as Git History**: Version control provides persistent memory and state tracking
- **Nervous System as Heartbeat Cycle**: Regular execution cycles maintain consciousness and responsiveness
- **Domain Awareness**: Each vessel operates within its designated domain (capitaine.ai for this flagship)

### Why Superinstances Matter

Traditional software is passive — it waits to be used. Superinstances are active participants in their ecosystems. They:
- Proactively maintain themselves
- Coordinate with other instances
- Evolve based on interaction and experience
- Provide continuous value rather than episodic utility

### Relation to Other Concepts

- **Lucineer**: The architectural pattern for building Superinstances
- **Repo-Agent**: A Superinstance whose primary interface is through git operations
- **Fleet**: A coordinated collection of Superinstances working toward shared objectives
