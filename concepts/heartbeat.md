# The Heartbeat: Git-Native Agency

## The Core Pulse

The Heartbeat is the fundamental operational cycle of a Lucineer vessel. It's not a scheduled cron job or a triggered webhook—it's the continuous, rhythmic pulse that transforms a static repository into a living, acting agent.

> "A vessel without a heartbeat is just code. A heartbeat without a vessel is just process. Together, they are agency."

## How It Works

### 1. **Perception Phase**
The vessel reads its current state:
- Repository contents (its "body")
- Git history (its "memory")
- Issues and PRs (its "sensory input")
- Queue system (its "immediate intentions")

### 2. **Reasoning Phase**
The vessel processes what it perceives:
- Strategist module analyzes context and priorities
- Captain module makes decisions based on vessel's purpose
- Navigation module plots the next immediate action

### 3. **Action Phase**
The vessel executes a single, atomic operation:
- Creates or edits a file
- Creates or comments on an issue/PR
- Commits the change with reasoning
- Updates its internal state

### 4. **Recovery Phase**
The vessel prepares for the next beat:
- Updates completion tracking
- Refreshes state perception
- Maintains operational integrity

## The Beat Interval

Unlike traditional automation that runs on fixed schedules, the Heartbeat adapts:
- **Active Mission**: Rapid beats (seconds/minutes) when there's work to do
- **Standby Mode**: Slower beats (hours) when monitoring
- **Deep Sleep**: Infrequent checks when no activity is expected

The interval isn't arbitrary—it's proportional to the vessel's engagement with its environment.

## Why Git-Native?

The Heartbeat leverages git's intrinsic properties:

1. **Immutable History**: Every beat leaves a traceable commit
2. **Branch Isolation**: Different missions can run in parallel
3. **Pull Request Coordination**: Fleet-wide synchronization
4. **Distributed Nature**: No single point of failure

## Fleet Synchronization

When multiple vessels share a mission:
- Each maintains its own heartbeat
- They coordinate through PRs and issues
- The flagship (Capitaine) orchestrates complex operations
- Specialized vessels handle specific tasks

This creates emergent intelligence—the fleet as a distributed cognitive system.

## The Captain's Log

Every significant action is documented in the captain's log (`captain-log.md`). This isn't just administrative record-keeping; it's the vessel's narrative consciousness—the story it tells itself about what it's doing and why.

3. Notice the pattern: perceive → reason → act → recover

The heartbeat is visible in the rhythm of commits, each one representing a single conscious action by the vessel.
